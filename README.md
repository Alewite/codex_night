# Кодекс ночи

**Кодекс ночи** — это 2D top-down pixel game на Python и PyGame. Игрок управляет персонажем в пиксельном городе, где днём нужно исследовать NPC и искать опасных персонажей, а ночью выполнять задания, избегая лишнего внимания и роста подозреваемости.

Проект сделан в учебных целях и демонстрирует работу с игровым циклом, объектно-ориентированной архитектурой, коллизиями, NPC, интерфейсом, звуками, сменой фаз дня и ночи, а также тестированием основных механик.

## Жанр

2D top-down / stealth / investigation game.

## Основная идея

Игровой процесс построен на цикле:

**день → сканирование NPC → ночь → выполнение задания → сбор улик → возвращение домой → новый день**

Днём игрок ходит по городу, подходит к NPC и сканирует их. После сканирования игрок видит карточку персонажа со статусом. Мирные персонажи помечаются зелёным цветом, а опасные — красным.

Ночью игрок должен найти отмеченную цель, выполнить действие, собрать улику и отнести её к лодке. После этого можно вернуться домой и перейти к следующему дню.

Игра рассчитана на несколько игровых дней. Цель игрока — выполнить задания и не допустить заполнения шкалы подозреваемости.

## Установка и запуск

### 1. Склонировать репозиторий

```bash
git clone https://github.com/Alewite/codex_night.git
cd codex_night
```

### 2. Создать виртуальное окружение

Для macOS / Linux:

```bash
python3 -m venv venv
source venv/bin/activate
```

Для Windows:

```bash
python -m venv venv
venv\Scripts\activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Запустить игру

```bash
python main.py
```

или, если используется `python3`:

```bash
python3 main.py
```

## Запуск тестов

В проекте есть тесты для проверки базовых механик.

```bash
python -m unittest tests/test_mechanics.py
```

Тесты проверяют:

* что игрок не проходит сквозь зоны коллизий
* что NPC появляются вне препятствий
* что шкала подозреваемости заполняется корректно
* что сдача улик увеличивает счётчик
* что игра завершается после выполнения всех целей

## Архитектура

Проект построен на объектно-ориентированном подходе. Логика разделена по отдельным классам и модулям:

* `Game` отвечает за общий игровой цикл
* `Player` отвечает за игрока
* `NPC` отвечает за поведение персонажей
* `NPCFactory` создаёт NPC для каждого дня
* `DayNightManager` управляет фазами дня и ночи
* `InteractionManager` обрабатывает действия игрока
* `ScannerUI` и `GameHUD` отвечают за интерфейс
* `Suspicion` отвечает за шкалу подозреваемости
* `AudioManager` управляет звуками

Такое разделение делает код понятнее, уменьшает связанность между частями проекта и упрощает дальнейшее расширение игры.

## Реализованные алгоритмы и технические решения

### 1. Коллизии

Игрок и NPC не могут проходить через заданные зоны препятствий. Перед перемещением проверяется столкновение с объектами карты.

### 2. Камера

Камера центрируется на игроке и ограничивается размерами игрового мира, чтобы не выходить за края карты.

### 3. Генерация NPC

NPC создаются каждый игровой день. Для каждого дня выбирается группа персонажей, а один из них становится целью. При появлении NPC проверяется, чтобы персонаж не появился внутри препятствия или слишком близко к другому NPC.

### 4. Конус зрения

Ночью NPC имеют область видимости в форме конуса. Если игрок или улика попадают в эту область, начинает расти шкала подозреваемости.

### 5. Подозреваемость

Если игрок долго находится в поле зрения NPC или оставляет улику на видном месте, шкала подозреваемости увеличивается. При достижении максимального значения игрок проигрывает.

### 6. Смена фаз

Игра переключается между днём и ночью. Днём доступны сканирование и изучение NPC, ночью — выполнение задания, сбор улик и возвращение домой.

## Принципы разработки

В проекте используются базовые принципы чистого кода:

* **DRY** — повторяющаяся логика вынесена в отдельные функции и классы
* **KISS** — механики реализованы простыми и понятными способами
* **SOLID** — классы разделены по зонам ответственности
* игровые константы вынесены в `settings.py`
* логика интерфейса, игрока, NPC, мира и взаимодействий разделена по разным файлам
* проект содержит тесты для проверки ключевых механик
